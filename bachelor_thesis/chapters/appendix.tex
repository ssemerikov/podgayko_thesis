\section{Опис кластерів VOSViewer}
\label{app:clusters}

Нижче наведено детальний опис 4~кластерів, ідентифікованих VOSViewer на~основі аналізу 28~ключових слів (поріг~$\geq$\,100).

\subsection*{Кластер~1: Системи та політика ПТО (VET~Systems and~Policy Framework)}

Кластер об'єднує 11~термінів і~описує інституційний та~системний вимір ПТО на~макрорівні. Центральний термін \textit{vet} (TLS\,=\,17\,857, входження\,=\,1\,448) є~ядром усієї мережі. \textit{Training} (TLS\,=\,13\,158, входження\,=\,965)~-- другий за~важливістю процесний термін. \textit{Country} (250), \textit{germany} (120), \textit{company} (158) вказують на~порівняльні дослідження національних систем, особливо дуальної моделі Німеччини та~ролі роботодавців. \textit{Development} (574) та~\textit{model} (380) свідчать про~фокус на~моделюванні та~розвитку систем ПТО. \textit{Higher education} (112)~-- зв'язок ПТО з~вищою освітою. Середній рік кластера~-- 2017,88 (найновіший). Тренд: від~порівняльних досліджень до~аналізу співпраці з~бізнесом.

\subsection*{Кластер~2: Емпіричні дослідження результатів навчання (Empirical Research on~Student Outcomes)}

Кластер містить 8~термінів і~відображає емпіричну дослідницьку парадигму. \textit{Student} (TLS\,=\,14\,844, входження\,=\,1\,161) та~\textit{study} (TLS\,=\,14\,924, входження\,=\,1\,188)~-- два найсильніші терміни, що~утворюють методологічне ядро. \textit{Assessment} (473), \textit{performance} (353), \textit{effect} (336)~-- тріада вимірювання результатів. \textit{Group} (296) та~\textit{difference} (208)~-- ознаки порівняльного експериментального дизайну. \textit{School} (512)~-- контекст навчання. Середній рік~-- 2017,41. Тренд: стабільне ядро кількісних досліджень.

\subsection*{Кластер~3: Педагогічна практика та~компетентнісний підхід (Pedagogical Practice and~Competence Development)}

Кластер об'єднує 6~термінів на~мезорівні. \textit{Teacher} (TLS\,=\,8\,600, входження\,=\,593)~-- центральний актор. \textit{Competence} (TLS\,=\,6\,352, входження\,=\,371)~-- концептуальне ядро. \textit{Practice} (423), \textit{learning} (342), \textit{work} (329), \textit{context} (280)~-- зв'язок із~реальним професійним середовищем (workplace~learning). Середній рік~-- 2017,63. Тренд: від~загальних компетентностей до~контекстуалізованого навчання.

\subsection*{Кластер~4: Когнітивні результати та~навички (Cognitive Outcomes and~Skills)}

Найменший, але~семантично щільний кластер з~3~термінів. \textit{Skill} (TLS\,=\,5\,910, входження\,=\,375)~-- центральний результат ПТО. \textit{Knowledge} (TLS\,=\,4\,652, входження\,=\,312)~-- когнітивна основа. \textit{Problem} (TLS\,=\,3\,324, входження\,=\,198)~-- проблемно-орієнтований підхід. Середній рік~-- 2017,40 (найкласичніший). Це~<<результатний>> кластер, що~описує цільові компетенції ПТО.

\subsection*{Д.2. Оцінкова рубрика (стислий зміст)}
\label{app:eval-rubric}

Рубрика складається з~4~розділів. Шкала Лікерта 1--5 (1~=~категорично~не~погоджуюсь; 5~=~категорично погоджуюсь). Усі~бальні пункти обов'язкові; коментарі~-- опційні.

\textbf{Розділ A. System Usability Scale} (10~стандартних пунктів за~Brooke 1996; нормалізація до~0--100 за~стандартною формулою, де~непарні~$=$~бал$-1$, парні~$=$~$5-$бал, сума~$\times 2{,}5$):

\begin{enumerate}[noitemsep,topsep=2pt,leftmargin=*]
    \item Я~думаю, що~користувався б~цією системою часто.
    \item Я~знайшов(ла) систему надмірно складною.
    \item Я~думаю, що~системою легко користуватися.
    \item Я~думаю, що~мені знадобилася б~технічна підтримка.
    \item Різні функції системи добре інтегровані.
    \item У~системі забагато непослідовності.
    \item Більшість людей швидко навчиться користуватися.
    \item Я~знайшов(ла) систему дуже громіздкою.
    \item Я~почувався(лася) дуже впевнено у~користуванні.
    \item Я~мав(ла) вивчити багато речей перед використанням.
\end{enumerate}

\textbf{Розділ B. Доменно-специфічні пункти} (8~пунктів з~опційним коментарем):

\begin{enumerate}[noitemsep,topsep=2pt,leftmargin=*]
    \item[B.1] Відповідність полів формам Specifikatsiya v2.0 та~реальній моніторинговій практиці.
    \item[B.2] Зрозумілість крос-формових попереджень (1.1$\leftrightarrow$3.1, 6.1 інваріанти оцінок).
    \item[B.3] Адекватність ролей та~територіальної прив'язки (директор / інспектор / держслужбовець).
    \item[B.4] Придатність агрегацій для~прийняття управлінських рішень.
    \item[B.5] Готовність обліковувати переміщені заклади (юридична $\neq$ фактична адреса; статус «евакуйований»; ABAC двох територій).
    \item[B.6] Готовність обліковувати ВПО та~ветеранів серед~здобувачів.
    \item[B.7] Очікувана надійність аудит-trail (SHA-256 хеш-ланцюг + \texttt{verify()}).
    \item[B.8] Імовірність впровадження у~Вашій установі.
\end{enumerate}

\textbf{Розділ C. Відкриті запитання} (вільний текст):

\begin{enumerate}[noitemsep,topsep=2pt,leftmargin=*]
    \item[C.1] Що~було найзручнішим? (1--3~моменти)
    \item[C.2] Що~викликало найбільші труднощі? (1--3~моменти)
    \item[C.3] Що~бракує для~реального впровадження? (1--3~функції / інтеграції)
    \item[C.4] Self-assessment готовності до~пілоту за~шкалою TRL 1--9 з~обґрунтуванням.
\end{enumerate}

    \item D.2 Регіон установи (Центральна / Західна / Південна / Східна Україна / Київ / інше).
    \item D.3 Стаж у~сфері (менше~3 / 3--5 / 6--10 / 11--20 / більше~20~років).
    \item D.4 Чи~представляє переміщений заклад (так~повністю / так~частково / ні / не~стосується).
    \item D.5 Технологічний рівень робочого місця (значний / помірний / базовий / обмежений).
    \item D.6 Згода на~half-day сесію (так / можливо / ні).
\end{itemize}

\subsection*{Д.4. Протокол half-day think-aloud сесії (стислий зміст)}
\label{app:eval-session}

Тривалість~-- 3,5--4~год з~двома перервами по~15~хв; формат~-- Zoom або~очно. Структура:

\begin{enumerate}[noitemsep,topsep=2pt,leftmargin=*]
    \item Знайомство та~налаштування ($\sim$15~хв)~-- підтвердження згоди, welcome-скрипт, зменшення «hawthorne effect».
    \item Сценарій~1 ($\sim$30~хв з~5--7~хв обговорення).
    \item Сценарій~2 ($\sim$30~хв).
    \item Перерва~1 (15~хв; не~обговорюємо систему).
    \item Сценарій~3 ($\sim$35~хв).
    \item Сценарій~4 ($\sim$35~хв).
    \item Перерва~2 (15~хв).
    \item Сценарій~5~-- кейс переміщеного закладу ($\sim$45~хв).
    \item Ретроспекція ($\sim$30~хв)~-- 5~питань: 6-міс перспектива, 3~найбільші проблеми, 3~найкращі речі, що~пропустив дослідник, вільне обговорення.
\end{enumerate}

Автор фіксує: час кожного «застрягання»~$>$~30~сек; дослівні цитати-вербалізації (особливо плутанини); severity-rating проблем за~Nielsen (1~=~косметична; 2~=~мінорна; 3~=~серйозна; 4~=~критична). Усі~discrete~issues з~ID \texttt{iss-EXP-<n>-<seq>} винесено у~CSV з~map'ом «severity~$\times$~модуль коду». Експерт заповнює формальну рубрику окремо \emph{після} сесії, щоб~уникнути впливу автора на~бали.

Контр-етичні запобіжники: участь добровільна і~може бути перервана у~будь-який момент; аудіозапис тільки за~окремою явною згодою (видаляється після~транскрибування або~не~пізніше 30~днів); автор-як-insider \emph{відкликається} при~помічанні власних направляючих питань і~фіксує це~у~нотатці; усі~питання конфлікту інтересів виносяться на~засідання з~науковим керівником.

\subsection*{Д.5. Цифрова форма та~OUTPUT-схема}
\label{app:eval-form}

\textbf{Google Form / LimeSurvey:} анонімна (без~збору email/IP), без~авторизації, з~можливістю редагування після~подання. Експорт у~CSV з~стандартизованою схемою у~36~стовпців: \texttt{respondent\_code, timestamp, sus\_a1..a10, b1\_score..b8\_score, b1\_comment..b8\_comment, c1\_text..c4\_justification, d1\_role..d6\_halfday, duration\_min, tech\_issues, tech\_issues\_text}. Збереження у~\texttt{expert\_eval/raw\_responses\_EXP-<n>.csv}~-- по~одному файлу на~експерта.

\textbf{Заповнюваний PDF:} генерується через~\texttt{pandoc 06\_rubric.md -o 06\_rubric\_fillable.pdf} з~додаванням form-fields (Adobe Acrobat); парсинг у~CSV~-- скрипт \texttt{scripts/eval\_pdf\_to\_csv.py} (залежність \texttt{pypdf~$\geq$~4.0}).

\textbf{Агрегація:} \texttt{scripts/eval\_aggregate\_experts.py} зчитує всі~\texttt{raw\_responses\_*.csv}, валідує схему, обчислює описові метрики (SUS нормалізована 0--100, B-блок 1--5, B-середнє, TRL-середнє), експортує: \texttt{responses\_anonymized.csv} (без~ідентифікаційних колонок), \texttt{results\_table.tex} (підвантажується у~підрозділ~\ref{sec:p2-eval-expert}), \texttt{summary\_stats.json} (машино-читаний звіт). При~$n{<}12$ скрипт виводить попередження і~маркує SUS-середнє як~«описовий пріор» (95\%~CI~$\sim\pm 25$~балів за~\cite{SauroLewis2016}).

\textbf{Якісне тематичне кодування} C-блоку та~B-коментарів виконується вручну (двократне читання автором + керівник у~ролі другого кодера), результати~-- у~\texttt{expert\_eval/summary.md} з~severity-mapped списком issues~$\to$~модулі коду~$\to$~рекомендована дія, інтегрується у~висновки до~підрозділу~\ref{sec:p2-eval-expert}.
