# Комплект експертного оцінювання (T2.6 / T2.7)

Каталог містить повний INPUT/OUTPUT-комплект для формального експертного оцінювання та поглибленого думко-вголос (think-aloud) дослідження прототипу інформаційно-аналітичної системи моніторингу й аналізу освітньої діяльності закладів П(ПТ)О.

## Призначення

Виконує задачі **T2.6** (підготовка матеріалів) і **T2.7** (проведення оцінювання) дорожньої карти кваліфікаційної роботи. Цільова кількість респондентів — $n\geq 3$ для формальної анкети; з-поміж них 1–2 — для half-day сесій.

## Структура каталогу

```
expert_eval/
├── README.md                            (цей файл)
│
├── INPUT (надається експертам)
│   ├── 01_invitation_letter.md          лист-запрошення
│   ├── 02_consent_form.md               форма інформованої згоди (GDPR-аналог)
│   ├── 03_walkthrough_script.md         сценарій walkthrough-відеозапису (~10–12 хв)
│   ├── 04_demo_credentials_template.md  шаблон облікових даних (заповнюється при розгортанні)
│   ├── 05_scenario_card.md              5 типових end-to-end сценаріїв
│   └── 06_rubric.md                     оцінкова рубрика (SUS + B + C + D)
│
├── PROTOCOL (для автора-дослідника)
│   ├── 07_session_protocol.md           протокол half-day think-aloud сесії
│   └── 08_form_schema.md                схема Google Form / LimeSurvey
│
└── templates/                           OUTPUT (шаблони, заповнюються після збору)
    ├── responses_template.csv           заголовок CSV для агрегації
    ├── results_table_template.tex       шаблон LaTeX-таблиці підсумків
    ├── summary_template.md              шаблон якісного звіту з тематичним кодуванням
    └── deep_session_NOTES_TEMPLATE.md   шаблон нотаток half-day сесії
```

## Підтримуючі скрипти

Розташовані у `bachelor_thesis/scripts/`:

- `eval_aggregate_experts.py` — агрегує CSV-відповіді у `responses_anonymized.csv`, обчислює описові SUS-метрики, експортує `results_table.tex`.
- `eval_pdf_to_csv.py` — fallback-парсер для заповнюваних PDF.

## Робочий процес

### Підготовка (T2.6, виконано 2026-05-10)

1. ✅ Підготовка усіх INPUT/OUTPUT-матеріалів цього каталогу.
2. ⏳ Розгортання seeded-instance прототипу (1 область, 3 райони, 10 громад, 5 закладів, з них 1 «евакуйований»). Статус — після узгодження середовища (on-prem НМЦ ПТО Луганщини або тимчасовий VPS).
3. ⏳ Запис walkthrough-відеозапису згідно зі сценарієм у `03_walkthrough_script.md` (тривалість 10–12 хв).
4. ⏳ Створення Google Form за схемою `08_form_schema.md` або генерація заповнюваного PDF на основі `06_rubric.md`.
5. ⏳ Узгодження списку експертів з керівником: ціль — 3 респонденти різного профілю (директор П(ПТ)О / інспектор обласного департаменту / представник МОН або науковець з педагогічної інформатики); 1–2 з них погоджуються на half-day сесію.

### Розсилка та збір (T2.7)

1. Email-розсилка з вкладенням `01_invitation_letter.md` (конвертовано у DOCX/PDF) + посилання на `02_consent_form.md` для електронного підпису або скан-повернення.
2. Після згоди — надсилання `03_walkthrough_script.md` (як супровідного тексту до відеозапису), `04_demo_credentials.md` (заповненого), `05_scenario_card.md`, посилання на форму або заповнюваний PDF з `06_rubric.md`.
3. Орієнтовний термін для заповнення — 7 робочих днів від отримання матеріалів.
4. Half-day сесія планується індивідуально; протокол — `07_session_protocol.md`. Тривалість — 3,5–4 години (15 хв вступ → 5 сценаріїв по 30–40 хв → 30 хв ретроспекція). Усі нотатки зберігаються в `deep_session_<expert_code>_notes.md`.

### Аналіз і інтеграція

1. CSV-відповіді консолідуються скриптом `eval_aggregate_experts.py` у `responses_anonymized.csv`.
2. Обчислюється описова SUS-статистика з $\pm$95% довірчим інтервалом (з явним застереженням про $n{<}12$).
3. B-блок та C-блок проходять якісне тематичне кодування (двократне читання автором + супервайзер у ролі другого кодера для перевірки міжкодувальної надійності за вибраними категоріями).
4. Результати думко-вголос сесій зводяться у мапу проблем «severity-rating × модуль коду».
5. Підсумки переносяться у `summary.md` та `results_table.tex`; інтегруються до тексту дисертації через `\input{expert_eval/results_table.tex}` у підрозділі 2.4.6.

## Етичні та правові аспекти

- Згода обробки персональних даних відповідно до Закону України «Про захист персональних даних» №2297-VI та принципів GDPR (мета збору, права суб'єкта, термін зберігання, контактні дані відповідального).
- Усі персональні ідентифікатори знеособлюються до коду `EXP-<n>` після завершення збору; зв'язний ключ не зберігається.
- Аудіозапис half-day сесій — лише за окремою явною згодою; за відсутності згоди ведуться лише письмові нотатки.
- Робота декларує конфлікт інтересів автора як заступника директора з НМР НМЦ ПТО Луганщини; його колеги виключаються зі списку респондентів формальної анкети, а беруть участь лише як стейкхолдери у half-day режимі за умови присутності супервайзера.

## Доменно-специфічні застереження

- Прототип валідовано лише на синтетичних даних (TRL 4); експертна оцінка — у режимі «scenario-based forecast», не «production-grade test». Це чітко комунікується у вступному параграфі рубрики.
- Серед сценаріїв обов'язково присутній кейс переміщеного закладу (юридична адреса в окупації, фактична — у приймаючій громаді) для валідації специфічних дизайн-рішень підрозділу 2.3.7.
